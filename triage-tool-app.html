Endoscopy Triage & Diagnosis Tool

Clinical Decision Support • R Shiny • Random Forest • Healthcare Analytics

Project Overview

A specialized Shiny web application designed to assist clinicians in diagnosing and triaging patients with gastrointestinal bleeding. The tool uses Random Forest machine learning models trained on clinical data to provide probabilistic predictions for bleeding source identification, urgent endoscopy necessity, and appropriate patient disposition. This clinical decision support system helps healthcare professionals make data-driven decisions while maintaining the primacy of clinical judgment.

Technical Stack

  • Programming Language: R (version 4.0+)
  • Web Framework: Shiny for interactive web applications
  • Machine Learning: randomForest package for Random Forest models
  • Data Processing: dplyr, readr for data manipulation
  • Visualization: ggplot2 for custom prediction plots
  • Deployment: rsconnect for Shiny server deployment
  • Data Management: pins package for model versioning

Clinical Prediction Models

  • Bleeding Source Prediction: Classifies GI bleeding as upper, mid, or lower gastrointestinal
  • Urgent Endoscopy Assessment: Determines the need for immediate endoscopic intervention
  • Patient Disposition: Recommends ICU vs. non-ICU placement based on severity
  • Resuscitation Needs: Evaluates immediate resuscitation requirements

Application Features

  • Interactive web interface for clinical data entry
  • Real-time probabilistic predictions using Random Forest models
  • Custom ggplot2 visualizations showing prediction confidence
  • Feature importance display for model interpretability
  • Physician diagnosis input fields for comparison and validation
  • Demographic and clinical parameter collection (hematemesis, melena, vital signs, lab values)
  • Compute and refresh functionality for multiple patient assessments
  • Stacked bar plots for easy interpretation of prediction results

Technical Architecture

  • app.R: Main Shiny application with UI and server logic
  • createModels.R: Random Forest model training pipeline (model_S, model_R, model_E, model_D)
  • makePredictions.R: Prediction functions for processing new patient data
  • displayPlots.R: Custom ggplot2 visualization functions for model outputs
  • train_data.csv: Clinical training dataset for model development

Workflow Process

  • Data Input: Clinicians enter demographic and clinical data through Shiny UI
  • Model Processing: Four Random Forest models process input data simultaneously
  • Probability Generation: Each model outputs probabilistic predictions for its specific outcome
  • Visualization: Results displayed as stacked bar plots with feature importance rankings
  • Clinical Integration: Predictions complement physician judgment for enhanced decision-making

Clinical Applications & Disclaimers

  • Decision Support: Provides probabilistic guidance for complex GI bleeding cases
  • Educational Tool: Demonstrates machine learning applications in clinical settings
  • Research Platform: Framework for developing additional clinical prediction models
  • Important Note: Intended as demonstration tool, not replacement for clinical judgment
  • Compliance: Users must ensure healthcare privacy law compliance with patient data
  • Validation: Not approved as medical device - requires clinical validation for deployment
